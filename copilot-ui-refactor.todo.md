# Desktop UI Consistency Refactor Todo List

## PHASE 0 – Baseline & Guardrails
- Insert and preserve this todo list
- Define scope, exclusions, and non-negotiable rules
- Inventory all current layouts, components, and styling sources
- Document risks and known issues

## PHASE 1 – Layout & Structure
- Standardize page-level layout patterns
- Unify spacing, padding, and grid systems
- Remove legacy layout hacks and inline styles
- Refactor to use only approved layout components

## PHASE 2 – Component Consistency
- Audit and refactor all card, table, button, sidebar, and header components
- Replace ad-hoc or page-specific components with shared versions
- Ensure all components use Tailwind-first styling
- Remove duplicate or inconsistent component variants

## PHASE 3 – Visual Consistency
- Standardize colour palette and typography
- Remove mixed or legacy colour usage
- Apply consistent border radius, shadows, and iconography
- Audit and fix any remaining visual inconsistencies

## PHASE 4 – QA & Polish
- Cross-browser and device testing (desktop only)
- Accessibility review (focus, contrast, keyboard nav)
- Final code cleanup and documentation
- Prepare for handoff or deployment
